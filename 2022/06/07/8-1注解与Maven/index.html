3.注解 - [重点理解]

3-1 注解简介

注解是什么 ?

Java注解是附加在代码中的一些元meta信息，用于一些工具在编译、运行时进行解析和使用，起到说明、配置xml的功能

注解本身不做任何处理，不会影响源代实际功能。只能通过注解解析程序读取，再进行相应其他扩展功能实现

Java 注解（Annotation）又称 Java 标注，是 JDK5.0 引入的一种注释机制。

Java 语言中的类、方法、变量、参数和包等都可以被标注

@Param比如Mybatis注解 -首字母大写 - 注解也是类型，按规范大写

@Override

注释 - Comment - 写给程序员看的

|-代码的说明

注解 - Annotation -写给程序 [注解解析程序 ]看的

  |-代码的说明

注解有什么作用？

作用分类：

①编写文档：通过代码里标识的元数据生成文档【生成文档doc文档】 @Documented

② 代码分析：通过代码里标识的元数据对代码进行分析【使用反射】 –自定义框架，注解解析程序来读取

③编译检查：通过代码里标识的元数据让编译器能够实现基本的编译检查【Override】
