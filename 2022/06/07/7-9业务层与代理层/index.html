3.AOP与代理模式 [重难点]

3-1 AOP

OOP - 面向对象编程 - Object Oriented Programming

AOP - Aspect 方面、切面 , Oriented 方向、面向， Programming

|- 面向切面编程 思想

AOP是Spring提供的关键特性之一 ，另一个特性是 IOC [依赖注入]

AOP即面向切面编程，是OOP编程的有效补充。使用AOP技术，可以将一些系统性相关的编程工作，独立提取出来，独立实现，然后通过切面切入进系统。

从而避免了在业务逻辑的代码中混入了很多的系统性相关的逻辑–比如：事务管理、日志记录、权限管理等，从而达到了将不同的关注点分离出来松耦合的效果

3-2 代理模式

AOP是一种面向切面编程编程思想，代理模式作为该思想的实现手段

AOP代理分为两种：

  • 静态代理

  |- 在编译期确定代理类与被代理类[目标Target]的关系

  • 动态代理 [灵活 - 反射]

    |- 在运行时动态的确定代理类与被代理类[目标Target]的关系
