3.线程优先级 [了解]

  • 优先级代表线程抢占到cpu资源的概率，不代表线程一定会优先执行

  • 可以通过线程 对象方法设置优先级，如果不设置，默认优先级为NORMAL普通

    • static int MAX_PRIORITY 线程可以拥有的最大优先级。
      static int MIN_PRIORITY 线程可以拥有的最小优先级。
      static int NORM_PRIORITY 分配给线程的默认优先级。

4.线程的常用方法-[重点]

关于线程 操作的绝大部份方法都封装在Thread类中

(1) sleep()与interrupt

sleep() –必须指定毫秒时间值

是一个静态方法 Thread.sleep()，必须使用try…catch处理InterruptException异常

interrupt() –中断休眠的线程，引发InterruptException异常

(2) yield()线程让步

Thread.yield() 线程的让步方法

作用：使得当前线程让出CPU资源，让具有相同优先级的线程来获取CPU的执行权，但是不是绝对，因为有可能没有比当前线程优先级高的线程，当前的让步线程会被再次选中执行.

(3) join() 线程抢占

让多个线程执行时实现同步的效果

在多线程应用中，有时需要将一个主干任务拆分成多个子任务

A线程抢占B线程的CPU资源，线程B必须要等到线程A的代码全部执行完了才能继续执行
