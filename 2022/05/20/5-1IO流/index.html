1.File类认识与操作

File -文件 Directory - 目录 对应于windows系统中的文件夹 [folder]

1-1 File类的定义

1-2 File对象的构造

    • File(File parent, String child) 从父抽象路径名和子路径名字符串创建新的 File实例。
      File(String pathname) 通过将给定的路径名字符串转换为抽象路径名来创建新的 File实例。
      File(String parent, String child) 从父路径名字符串和子路径名字符串创建新的 File实例。

1-3 File对象常用API

  • booleanexists() 测试此抽象路径名表示的文件或目录是否存在
  • boolean mkdir() 创建目录 、 mkdirs() -创建目录及子目录
  • String getAbsolutePath() 返回此抽象路径名的绝对路径名字符串
  • boolean createNewFile()` 创建一个新的文件
  • booleandelete() 删除一个文件
  • StringgetName() 获取文件的名称或目录的名称
  • String getParent()` 获取父级目录的名称
  • File getParentFile() 获取父级目录的File实例
  • boolean isFile() 测试是否是文件
  • boolean isDirectory() 测试是否是目录
  • long lastModified() new Date(long) – 距1970.1.1以来的毫秒值 抽象路径名表示的文件上次修改的时间
  • long length() 返回由此抽象路径名表示的文件的长度 [大小] 单位字节Byte
  • File[] listFiles() 获取由该抽象路径名表示的目录中的文件(子目录)

2.IO流

IO流的定义

IO - Input/ Output 输入、输出

流 - stream – 数据序列 【01二进制位】

  • 我们能通过File实例可抽象表示OS系统中的某个文件或目录，但是我们不能进行对File进行读取操作？
    • 要实现该操作需要借助于IO流
  • IO流里面的四大抽象超类
    • xxxStream 通常都代表的是一个字节流
    • xxxReader / xxxWriter 通常都表示一个字符流 【读入写出】

2-4 字符流

  • 当使用字节流 读取文本字符的文件中，只要存在中文汉字，免不了，会出现乱码
  • 字符流是按照文本的字符来读取和输出
  • 字符流= 字节流+编码表
  • java.io 提供字符流，为了针对文本字符文件的读写方便

2-5 缓冲流

  • 又称为高级流[处理流]

  • 缓存是IO的一种性能优化，缓存流为IO流增加了内存缓存区，有了缓存区，便可以减少系统直接与硬盘交互传输的次数，增加每次传送的数据量，借以提高传输效率，保护硬件

  • 自带8K缓冲区

  • BufferedInputStream 缓冲输入字节流 /BufferedOutputStream()缓冲输出字节流

  • BufferedReader缓冲输入字符流 / BufferedWriter 缓冲输出字符流
